# 02 コーディング & アルゴリズム対策

**優先度：** ★（ML/AIエンジニアとしてPythonは得意なはず）
**形式：** 制限時間内でのDSA問題解答 + 思考プロセスの説明

---

## このラウンドで見られること

| 評価軸 | 内容 |
|---|---|
| 問題解決 | 見知らぬ問題を分解してアプローチを組み立てる力 |
| コード品質 | 正確・クリーン・読みやすいコード |
| 計算量分析 | 時間・空間計算量のトレードオフを説明できるか |
| コミュニケーション | 書きながら思考を声に出して説明できるか |
| エッジケース | 指摘される前に境界条件を考慮できるか |

---

## 重点テーマ

Google FDEのコーディングラウンドで頻出のパターン：

- **文字列・配列操作** — スライド窓、二ポインタ、ハッシュマップ
- **木・グラフ探索** — BFS/DFS、再帰
- **動的計画法** — メモ化、ボトムアップ
- **ヒープ・優先度キュー** — Top-K問題
- **再帰・バックトラッキング**

---

## 練習方針

1. LeetCode（Medium中心）を毎日1〜2問
2. 解く前に必ず「方針を声に出してから実装」する習慣をつける
3. 解いた後は時間・空間計算量を必ず分析する
4. エッジケース（空リスト・負数・重複）を確認する
