# 05 Googleyness（行動面接）対策

**優先度：** ★★
**形式：** 過去の経験ベースの行動面接（STAR形式）

---

## このラウンドで見られること

FDEは顧客環境に入り込む役割のため、行動面接の比重が通常のSWEより高い。

| 評価軸 | 内容 |
|---|---|
| オーナーシップ | 失敗を含めて結果に責任を取れるか |
| 曖昧さへの対応 | 道筋が見えないとき、どう動くか |
| コラボレーション | エンジニア以外と、権限なしに影響を与えられるか |
| 謙虚さ | 失敗・間違いを正直に振り返れるか |
| インパクト | 活動ではなく、測定可能な成果を語れるか |

---

## STAR フォーマット

```
Situation（状況）  — どんな文脈・背景だったか（1〜2文）
Task（課題）       — 自分が担うべき役割・責任は何だったか（1文）
Action（行動）     — 自分が具体的に何をしたか（メイン・3〜4文）
Result（結果）     — どんな成果が出たか（数値化できるなら必ず入れる）
```

各ストーリーは **2〜3分以内** に収める。

---

## 準備すべきストーリーカテゴリ

| カテゴリ | 必要数 | ファイル |
|---|---|---|
| 曖昧な状況下での意思決定・行動 | 1本 | `story_ambiguity.md` |
| 失敗・挫折とそこからの学び | 1本 | `story_failure.md` |
| 権限なしに他者を動かした経験 | 1本 | `story_influence.md` |
| 技術的な難題を解決した経験 | 1本 | `story_technical.md` |
| チームをまたいで協力した経験 | 1本 | `story_collaboration.md` |

合計5本用意すると、どんな質問にも対応できる。

---

## よく聞かれる質問例

- 不完全な情報の中で問題に取り組んだ経験を教えてください
- 失敗したプロジェクトと、そこから学んだことを教えてください
- 権限なしに結果を出した経験を教えてください
- チームメンバーと意見が対立したときの対処法を教えてください
- 最もインパクトのあった仕事を教えてください

---

## 注意点

- 「チームが〜した」ではなく「**私が**〜した」を主語にする
- 曖昧な結果（「うまくいった」）ではなく**数値や具体的な変化**を入れる
- ネガティブな経験（失敗）を必ず1本用意する（正直さと学習能力を示す）
- FDE文脈では「顧客との信頼構築」「非エンジニアへの説明」エピソードが特に刺さる
