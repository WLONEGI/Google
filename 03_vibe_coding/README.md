# 03 Vibe Coding 対策

**優先度：** ★★
**形式：** 曖昧な要件から実際に動くものをコラボラティブに構築するセッション

---

## このラウンドで見られること

| 評価軸 | 内容 |
|---|---|
| 曖昧さへの対応 | 要件が不明確なとき、どう確認・仮定してから進めるか |
| 実践的判断力 | 最適解より「動くものを先に作る」実用的な判断ができるか |
| コード構造 | 時間プレッシャー下でも読みやすく保守しやすいコードを書けるか |
| デバッグ力 | 問題が出たときの特定・修正アプローチ |
| コミュニケーション | 判断の都度、面接官に思考を共有できるか |

---

## DSAラウンドとの違い

| | コーディング & アルゴリズム | Vibe Coding |
|---|---|---|
| 要件 | 明確なプロンプト | 曖昧・不完全 |
| 評価 | アルゴリズムの最適性 | 実装判断・コラボ力 |
| スタイル | 競技プログラミング的 | 実務コーディング的 |
| 途中変更 | なし | 仕様変更あり |

---

## 練習方針

1. **曖昧なお題から始める練習** — 「RAGのシンプルな実装を作って」「エージェントのログ集計ツールを作って」など
2. 最初に必ず **確認すべき点を3つ挙げてから** 実装に入る
3. 実装途中で仕様変更が来たときの対応練習
4. コードを書きながら声に出して説明する（ペア練習推奨）

---

## 練習テーマ例

- シンプルなRAGパイプラインを書く（ベクトル検索なしのBM25版でも可）
- LLM APIを呼ぶエージェントのツールループを実装する
- ログファイルを解析してトークン使用量を集計するスクリプト
- チャット履歴を管理するステートマシンを実装する
